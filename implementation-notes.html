Baseline

  • Repository begins on main at 27d4784 with user-authored uncommitted changes in the v1 report/brief path. Those changes are in scope for replacement; unrelated research_writeup.html remains untouched.
  • The original system is a 1,654-line Python package centered on a broad v1 Payload. Its validate command mutates the payload, calculates sizing, and appends calls.csv.
  • Baseline mypy reports 12 pre-existing errors, primarily untyped third-party boundaries plus local annotation issues.
  • System Python 3.13.5 segfaults inside pytest capture initialization before project code runs.

Decisions

  • Implement only interventions #1–#5; keep later scoring, database, blind-underwriting, and portfolio ideas out of this cutover.
  • Use typed JSON records and filesystem modules as the canonical implementation. The local scale does not justify a database projection.
  • Use explicit manual source qualification during the transition. All migrated sources begin probationary.
  • Preserve raw files in runs/ and canonicalize legacy evidence separately. Remove the derived v1 payload, call ledger, technical snapshot, and report.

Deviations

D-001 · Test runtime

Plan assumption: use the workspace default Python. Observed edge case: Python 3.13.5 segfaults in pytest's capture initialization, before test collection reaches repository code. Conservative choice: run the suite serially in an isolated Python 3.12.13 virtual environment at /tmp/thesis-momentum-venv.sNShFV. Production compatibility remains Python 3.10+.

D-002 · Editable installation

Plan assumption: install the package with its dev extra. Observed edge case: setuptools rejected the repository because package discovery was unspecified and multiple top-level directories existed. Conservative choice: add explicit package discovery to pyproject.toml. Editable installation now succeeds.

D-003 · Raw trailing whitespace

Plan assumption: run diff hygiene uniformly over the cutover. Observed edge case: the original supplement contains trailing spaces, and its canonical raw copy must remain byte-identical. Conservative choice: preserve those bytes, exclude only records/raw/** from the whitespace check, and verify the original/copy SHA-256 digests instead. No authored source or documentation file is excluded.
